## 代表專案

### EtherCAT 與 RS485 分散式 I/O 混和通訊系統開發
**期間**：2023/09 ~ 2025/11

#### 專案簡述
產學合作專案，主要負責：
- EtherCAT Slave 韌體整合
- 雙核心 SoC 架構下的 STM32 韌體開發
- 通訊流程設計
- 系統整合測試與除錯驗證

#### 專案成就
- 完成 EtherCAT PDO / SDO 資料對映與 cyclic / acyclic 通訊流程設計
- 針對 RS485 無固定通訊協定特性，依專案需求設計通訊協定與資料流程
- 整合週期與非週期資料傳輸，並於多點 I/O 輪詢條件下達成毫秒級通訊週期
- 導入 DMA buffering、State Machine、CRC / ACK / timeout 等可靠度機制，提升通訊穩定性
- 完成 I/O 二重化切換流程
- 整合驗證數位、類比、脈波、溫度與 HART 等多種類型 I/O 模組

#### 使用技術
- **程式語言**：C
- **開發平台**：Visual Studio、STM32CubeIDE、TwinCAT3

## 自傳

### 學歷背景與求職方向
我畢業於長庚大學電機工程研究所，求職方向以韌體工程師 / Embedded Firmware Engineer 為主。研究所期間主要聚焦於 C / STM32 嵌入式韌體開發，並累積工業通訊、系統整合與除錯驗證相關實務經驗。希望未來能投入 MCU 韌體、工業通訊、嵌入式系統或控制系統相關工作，持續在實務開發與整合驗證中深化能力。

### 重點專案與技術經驗
研究所期間，我參與產學合作專案，主要負責 EtherCAT Slave 韌體整合、雙核心 SoC 架構下的 STM32 韌體開發，以及通訊流程設計、系統整合測試與除錯驗證。專案中完成 EtherCAT PDO / SDO 資料對映與 cyclic / acyclic 通訊流程設計，並針對 RS485 無固定通訊協定的特性，依專案需求設計通訊協定與資料流程，將週期與非週期資料整合於同一架構中，在多點 I/O 輪詢條件下達成毫秒級通訊週期。同時，我也導入 DMA buffering、State Machine 與 CRC / ACK / timeout 等可靠度機制，並完成 I/O 二重化切換流程與多種類型 I/O 卡的整合驗證，涵蓋數位、類比、脈波、溫度與 HART 相關模組。

### 除錯驗證與工作方式
在開發與整合過程中，我習慣搭配示波器、邏輯分析儀、Wireshark、Docklight、J-Link / ST-Link 等工具進行封包分析、時序除錯與穩定性測試，協助定位問題並確認系統行為是否符合預期。除了技術實作外，我也重視問題追蹤與協作流程，會透過 checklist、版本控管與測試紀錄整理異常狀況與驗證結果，讓問題收斂與跨團隊溝通更有效率。這樣的工作方式也讓我在面對系統整合與通訊問題時，能較有條理地分析原因並持續改善。

### 未來方向與期望
我對韌體與底層系統開發有高度興趣，也希望未來能持續投入 MCU 韌體、嵌入式系統、工業通訊或控制相關領域。除了既有的 STM32、EtherCAT、RS485 與系統整合經驗外，我也持續補強 RTOS、CAN 等相關知識，期待在未來工作中累積更完整的產品開發、整合測試與除錯驗證能力，成為能獨立處理開發與問題排查的韌體工程師。
